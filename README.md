# Лабораторная работа №3: Управление памятью и умные указатели

## 1 - Утечка памяти (ex1.cpp)

Создала массив через new и "забыла" его удалить (не освободила память)
В результате:
- После выхода из функции все указатели теряются
- Выделенная память становится недоступной для освобождения 

## 2 - Использование std::unique_ptr (ex2.cpp)
Изменила ex1.cpp
В результате:
- std::unique_ptr автоматически освобождает память при выходе из области видимости
- Сделала класс для файлов, они закрываются автоматически

## 3 - Использование std::shared_ptr (ex3.cpp)
Создала ресурс с двумя указателями
В результате:
- Объект удалится, когда оба перестанут использовать
- use_count() показывает текущее количество владельцев ресурса
- Когда ptr2 уничтожается, счётчик уменьшается
- Объект автоматически удалится при выходе и ptr1 из области видимости
- Ресурс освобождается только когда счетчик достигает нуля

## 4 - Использование std::weak_ptr (ex4.cpp)
Создала ресурс с двумя указателями (shared - владелец и weak - наблюдатель)
В результате:
- weak_ptr используется для наблюдения за ресурсом
- Через weak_ptr проверяем жив ли объект

## Вывод по указателям

unique_ptr - Один владелец - Для обычных объектов
shared_ptr - Несколько владельцев - Когда нужно разделять владение над ресурсом
weak_ptr - Только наблюдение - Для проверки (не удален ли)

Лучше использовать умные указатели unique_ptr/shared_ptr, чтобы избежать утечек памяти.
